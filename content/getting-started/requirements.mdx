# Requirements

WP Bones requires **PHP ≥ 8.1**, **git**, **Composer**, **Node.js** with a package manager
(**yarn**, npm or pnpm), and optionally **WP-CLI**. The `php bones` command must run inside a
working WordPress install — a local dev environment like Laravel Valet, Local, Lando, Docker,
or WordPress Playground all work.

## Developer tools

If you are new to [Composer](https://getcomposer.org/), [Node.js / yarn](https://yarnpkg.com/), or
[webpack](https://webpack.js.org/), take a quick look at their docs before diving in.

- PHP ≥ 8.1
- [git](https://git-scm.com/)
- [Composer](https://getcomposer.org/)
- [Node.js 18+](https://nodejs.org/) with [yarn](https://yarnpkg.com/) (recommended — v2
  boilerplates ship a committed `yarn.lock`), [npm](https://www.npmjs.com/) or
  [pnpm](https://pnpm.io/)
- [WP-CLI](https://wp-cli.org/#installing) (optional but recommended)

<Callout>
  WP-CLI is optional, but the `make-pot` and `make-json` yarn scripts shell out to it for
  translation file generation — install it if you plan to translate the plugin.
</Callout>

## Supported operating systems

macOS, Linux, and Windows 10/11 are all supported. On Windows we recommend WSL2 for a smoother
PHP / Node toolchain experience.

## WordPress environment

`php bones` must be executed from a working WordPress install because it loads WordPress to
resolve plugin context, options, database connection, and so on. See
[WordPress environments](/docs/getting-started/wordpress-environments) for the setups we recommend
(Valet, Local, Lando, Docker, Playground, Vagrant…). Make sure `git` and `composer` are available
in the same shell where you run `php bones`; Node.js / yarn can live on the host machine.
